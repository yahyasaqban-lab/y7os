🏠 Introduction

Y7 OS is a Linux distribution built for running AI locally. It transforms any computer into a private AI workstation without requiring cloud services, subscriptions, or sending data externally.

🔒

100% Local

All AI processing happens on your device. No data leaves your computer.

💻

Low Resource

Runs on systems with as little as 4GB RAM.

🔌

Offline Capable

Works without internet after initial setup.

🌐

Bilingual

Full English and Arabic support.

System Requirements

Component Minimum Recommended
RAM 4 GB 8 GB+
Storage 20 GB 50 GB+
CPU 64-bit (x86/ARM) 4+ cores
GPU Not required NVIDIA (optional)

📥 Installation

Option 1: Bootable USB (Recommended)

Step 1: Download the ISO

Get the latest release from GitHub:

https://github.com/yahyasaqban-lab/y7os/releases/latest

Step 2: Flash to USB

sudo dd if=y7os-0.4.0-amd64.iso of=/dev/sdX bs=4M status=progress

Or use: Balena Etcher, Rufus, or Ventoy

Step 3: Boot and Login

Username: y7
Password: y7

Option 2: Quick Install Script

Install Y7 OS tools on any existing Linux system:

curl -fsSL https://raw.githubusercontent.com/yahyasaqban-lab/y7os/main/tools/y7-install | sudo bash

Option 3: Virtual Machine

QEMU

qemu-system-x86_64 -m 4G -cdrom y7os-0.4.0-amd64.iso -boot d -enable-kvm

y7-ai

Core Tool

Interactive AI chat from the command line.

Usage

y7-ai [options] [question]

Options

OptionDescription
--model, -mSpecify model (e.g., llama3.1:8b)
--system, -sSet system prompt
--lang, -lLanguage (en or ar)
--temp, -tTemperature (0.0-2.0)

Examples

y7-ai "what is kubernetes?"

y7-ai -m qwen2.5:7b "explain machine learning"

cat code.py | y7-ai "review this code"

y7-ai --lang ar "ما هو الذكاء الاصطناعي؟"

y7-models

Core Tool

Download, list, and manage AI models.

Commands

CommandDescription
listShow downloaded models
recommendSuggest best model for your RAM
download <model>Download a model
remove <model>Delete a model
search <query>Search available models

Examples

y7-models list

y7-models recommend

y7-models download qwen2.5:7b

y7-status

Core Tool

View system status and AI stack health.

Options

OptionDescription
--compact, -cSingle-line output
--watch, -wContinuous update
--json, -jJSON output

Examples

y7-status

y7-status --watch

y7-status --json | jq '.ram_used'

y7-gpu

Advanced

Detect and configure GPU acceleration.

Commands

y7-gpu detect          # Check for GPU
y7-gpu install nvidia   # Install NVIDIA drivers
y7-gpu status           # Show GPU status
✅
Supported: NVIDIA, AMD (amdgpu), Intel (i915)

y7-rag

Advanced

Ask questions about your documents using RAG (Retrieval Augmented Generation).

Workflow

y7-rag init                    # Initialize
y7-rag add ./documents/        # Add documents
y7-rag query "what is the conclusion?"  # Ask questions

Supported formats: .txt, .md, .pdf, .docx

y7-voice

Advanced

Talk to AI using voice input and output.

Commands

y7-voice install     # Install voice components
y7-voice chat        # Interactive voice chat
y7-voice speak "Hello"  # Text to speech
y7-voice transcribe recording.wav  # Transcribe audio

y7-serve

Advanced

Run AI as an API server for external access.

API Endpoints

ServiceURL
Ollama APIhttp://localhost:11434
OpenAI Compatiblehttp://localhost:11434/v1
Open WebUIhttp://localhost:3000
 
y7-serve status    # Check status
y7-serve webui     # Start Open WebUI

y7-update

System

Keep Y7 OS and models up to date.

y7-update check    # Check for updates
y7-update all      # Update everything
y7-update tools    # Update y7-* tools only
y7-update models   # Update models only

y7-backup

System

Backup and restore your configuration and models.

y7-backup create           # Backup config
y7-backup create --models  # Include models (large!)
y7-backup list             # List backups
y7-backup restore backup.tar.gz  # Restore

y7-persist

System

Save data across reboots when running from USB.

y7-persist status   # Check persistence
y7-persist setup    # Configure persistence

🔧 Troubleshooting

"No model found"

sudo systemctl start ollama
y7-models download tinyllama

"Connection refused"

sudo systemctl restart ollama
curl http://localhost:11434/api/tags

"Out of memory"

y7-models download tinyllama  # Use smaller model
systemctl status zramswap      # Check ZRAM

Open WebUI not loading

docker restart open-webui
docker logs open-webui
